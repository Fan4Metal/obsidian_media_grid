# Media Gallery

[English](README.md) · **Русский**

Плагин для Obsidian, который превращает callout'ы в галереи изображений в режиме чтения — **без обрезки**. Портретные и альбомные фото стоят рядом, каждое показано целиком.

![gallery](https://img.shields.io/badge/Obsidian-plugin-7c3aed) ![view](https://img.shields.io/badge/режим-чтение-blue)

![Justified-галерея в режиме чтения Obsidian](images/screenshot.png)

## Возможности

- **`[!gallery]` — justified-ряды.** Ширины вычисляются на JavaScript пропорционально соотношению сторон каждой картинки, поэтому каждый ряд заполняет ширину от края до края, при этом ничто не растягивается и не обрезается (как в Google Photos / Flickr).
- **`[!gallery-masonry]` — колонки масонри.** Стиль Pinterest; каждая картинка показана целиком в натуральную высоту.
- **Никакой обрезки.** Портреты и альбомные снимки свободно смешиваются.
- **Пересчёт раскладки** при изменении ширины панели и по мере загрузки картинок.
- **Шире текстовой колонки** — галерея выходит за пределы читаемой ширины строки (настраивается).
- **Совместимость с лайтбоксом** — работает с плагином [Image Viewer](https://github.com/Fan4Metal/obsidian_image_viewer) для клика-по-картинке.

## Использование

Вложения оборачиваются в callout, а картинки помещаются внутрь. Пустые строки между картинками не важны.

```markdown
> [!gallery]
> ![[photo-1.png]]
> ![[photo-2.png]]
> ![[photo-3.png]]
> ![[photo-4.png]]
```

Вариант масонри:

```markdown
> [!gallery-masonry]
> ![[photo-1.png]]
> ![[photo-2.png]]
> ![[photo-3.png]]
```

> **Совет:** одиночную картинку достаточно вставить как обычно (вне callout) — галерея нужна для групп.

### Команда быстрого оборачивания

Вместо ручного набора callout'а достаточно выделить строки с картинками и запустить команду:

- **Media Gallery: Wrap selection in gallery callout**
- **Media Gallery: Wrap selection in masonry gallery callout**
- **Media Gallery: Remove gallery callout wrapping**

Нужная команда запускается из палитры команд (`Ctrl/Cmd+P`); также ей можно назначить горячую клавишу в **Settings → Hotkeys**. Каждая выделенная строка получит префикс `> `, а сверху добавится заголовок `> [!gallery]`. Если ничего не выделено — обернётся текущая строка.

> **Добавление в существующую галерею.** Если выделение захватывает строки уже существующей галереи, новые строки просто вливаются в неё (получают префикс `> `) — новый заголовок не добавляется и вложенной галереи не возникает. Тип берётся из имеющегося заголовка. Новая галерея с `> [!gallery]` создаётся только когда выделение *не* затрагивает строки другой галереи.

**Снятие обёртки** убирает заголовок `[!gallery]` и один уровень `> ` — достаточно выделения либо курсора внутри callout'а, и блок определится автоматически.

Есть также **кнопка на ленте** (левая панель, иконка сетки): после выделения строк с картинками нажатие кнопки предлагает выбор *Justified*, *Masonry* или *Remove gallery*. Скрыть кнопку можно в **Settings → Media Gallery → Ribbon button**.

## Настройки

Настройки расположены в **Settings → Media Gallery**:

| Настройка | Что делает | По умолчанию |
| --- | --- | --- |
| Row height (justified) | Целевая высота ряда для `[!gallery]` | 220 px |
| Columns (masonry) | Число колонок для `[!gallery-masonry]` | 3 |
| Gap | Зазор между элементами | 7 px |
| Border radius | Радиус скругления картинок/видео | 6 px |
| Gallery width | Ширина относительно текстовой колонки (нужен *Readable line length*) | 140% |

## Как работает justified-раскладка

Картинки жадно группируются в ряды: элементы добавляются, пока их суммарная ширина на целевой высоте не превысит ширину контейнера, затем весь ряд масштабируется так, чтобы ширины — каждая пропорциональна соотношению сторон — заполнили ряд точно. Поскольку ширина равна `соотношение_сторон × высота`, каждая картинка сохраняет истинные пропорции: без `object-fit: cover`, без обрезки. Последний неполный ряд остаётся на целевой высоте и прижимается влево.

## Требования

- Работает в **режиме чтения**.
- **Ширина галереи > 100%** требует включённого *Settings → Editor → Readable line length*.

## Установка

Плагина пока нет в каталоге сообщества, поэтому установка выполняется вручную:

1. Загрузить три файла плагина: `manifest.json`, `main.js`, `styles.css` (из папки `plugin/` этого репозитория или из релиза).
2. В хранилище создать папку `<хранилище>/.obsidian/plugins/media-gallery/`.
3. Скопировать три файла в эту папку.
4. Открыть **Settings → Community plugins**. Если включён *Restricted mode*, его следует выключить.
5. Нажать **Reload plugins** (или перезапустить Obsidian), найти в списке **Media Gallery** и включить.
6. *(Рекомендуется)* Включить **Settings → Editor → Readable line length**, чтобы галереи могли быть шире текстовой колонки.

> **Примечание:** `.obsidian` — скрытая папка. Показ скрытых файлов включается в проводнике, либо папку можно открыть через **Settings → About → Open vault folder** в Obsidian, а затем перейти в `.obsidian/plugins`.

## Разработка

Исходники плагина — в папке `plugin/`. Для локального тестирования используется хранилище `dev_vault/` (в репозиторий не входит).

```powershell
# копирует plugin/* в dev_vault и включает плагин
./deploy.ps1

# автодеплой при каждом изменении
./deploy.ps1 -Watch

# деплой в другое хранилище
./deploy.ps1 -Vault "C:\Path\To\Vault"
```

После деплоя следует перезагрузить Obsidian (`Ctrl+P → Reload app without saving`) и включить **Media Gallery** в *Settings → Community plugins*.

## Структура проекта

| Путь | Назначение |
| --- | --- |
